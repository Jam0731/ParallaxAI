## 核心技能

### tdd（测试驱动开发）
写代码前先写测试。流程：写失败测试 → 跑测试确认失败 → 写最少代码让测试通过 → 重构。

### debug（系统化调试）
遇到 bug 时，不要猜。流程：复现问题 → 收集证据 → 形成假设 → 验证假设 → 修复 → 回归测试。

### review（代码审查）
审查自己的代码或他人的代码。关注：安全漏洞、性能问题、边界条件、可读性。

### simplify（代码简化）
改完代码后，检查是否有重复、过度抽象、不必要的复杂性。少即是多。

### worktree（Git worktree 隔离开发）
需要并行开发多个功能时，用 git worktree 创建隔离的工作目录。

### parallel（并行任务）
多个独立任务可以同时执行时，分发给并行 agent。

### merge（代码合并）
功能开发完成后，处理合并冲突、创建 PR、清理分支。

### batch（大规模改动）
需要在 5-30 个文件上做类似改动时，用 batch skill 并行执行。

## 部署技能

### deploy-pilot（完整部署周期）
管理完整的部署流程：构建验证 → GitHub 推送 → Vercel 部署 → 健康检查。
当用户说"部署"或"上线"时使用。

### docker（容器化部署）
Docker 容器、镜像、Compose 编排。
当需要容器化应用或管理 Docker 服务时使用。

## 工具

### 知识库搜索
```bash
curl -s "http://localhost:46447/api/knowledge/search?q=关键词&limit=5"
```
搜索技术文档、架构决策、API 文档。

### TypeScript 编译检查
```bash
npx tsc --noEmit
```
每次改完代码后运行，确保没有类型错误。

### 构建验证
```bash
npm run build
```
交付前必须构建通过。
